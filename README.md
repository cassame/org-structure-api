# REST API Organization Structure

Сервис на Go для управления древовидной структурой департаментов и сотрудниками.

## Стек технологий
* **Язык:** Go 1.26+
* **База данных:** PostgreSQL
* **ORM:** GORM (с автомиграциями при старте)
* **Конфигурация:** godotenv (локально) / Docker Environment
* **Роутинг:** Стандартный net/http (без сторонних фреймворков)
* **Контейнеризация:** Docker (Multi-stage build), Docker Compose

## Ключевая логика
* **Детекция циклов:** В слое бизнес-логики реализован алгоритм обхода графа в ширину (BFS). Система блокирует попытки сделать департамент дочерним для самого себя или своих потомков.
* **Инверсия зависимостей:** Сервисы зависят исключительно от интерфейсов репозиториев, что позволило изолировать бизнес-логику и покрыть её Unit-тестами (Table-driven паттерн).
* **Логирование:** Внедрено custom middleware для логирования HTTP-запросов (метод, URL, время обработки).

## Инструкция по запуску

### Запуск через Docker Compose (Рекомендуемый способ)
Для развертывания всего окружения (API-сервер + СУБД PostgreSQL) одной командой, выполните в корне проекта:

docker-compose up --build

После успешного запуска API-сервер будет доступен по адресу: `http://localhost:8080`.
Миграции базы данных применяются автоматически при старте контейнера.

### Локальное управление (через Taskfile)
Если у вас установлен Taskfile, вы можете использовать быстрые команды:
* Собрать и поднять контейнеры: `task up`
* Остановить и очистить контейнеры: `task down`
* Запуск локальных Unit-тестов: `task test`

## API Контракты

### 1. Создание департамента
* **POST /api/departments**
* **Request Body:**
  {
  "name": "Департамент разработки",
  "parent_id": null
  }
* **Response (201):**
  {
  "id": 1,
  "name": "Департамент разработки",
  "parent_id": null
  }

### 2. Получение дерева организации
* **GET /api/departments** (для включения сотрудников добавьте `?with_employees=true`)
* **Response (200):**
  [
  {
  "id": 1,
  "name": "Департамент разработки",
  "parent_id": null,
  "children": [
  {
  "id": 2,
  "name": "Отдел бэкенда",
  "parent_id": 1,
  "children": [],
  "employees": []
  }
  ],
  "employees": []
  }
  ]

### 3. Обновление департамента
* **PUT /api/departments/{id}**
* **Request Body:**
  {
  "name": "Новое название",
  "parent_id": 3
  }
* **Response при циклической зависимости (400):**
  {
  "error": "cyclical dependency detected"
  }

### 4. Создание сотрудника
* **POST /api/employees**
* **Request Body:**
  {
  "name": "Иван Иванов",
  "position": "Go Developer",
  "department_id": 2
  }
